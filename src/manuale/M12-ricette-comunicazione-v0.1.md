# M12 — Ricette: comunicazione e promozione

**Codice:** M12 · **Versione:** 0.1 · 13/07/2026 · Parte III · con le regole del Project Comunicazione (→ M5 §2) sempre attive

## R1 · Save the Date

**Template**
```
Prepara il testo del Save the Date per [EVENTO: edizione, date, sede]: main topics ([3–5 TEMI]), una riga di posizionamento (storia/numeri dell’evento), «programma completo in arrivo», indicazioni iscrizioni [CANALE]. Due formati: a) mail breve, b) testo per grafica (max 40 parole). Niente nomi di relatori non confermati.
```
**Attenzioni:** è la decisione tipica della riunione di programmazione (cfr. verbale d’aula): si esce coi main topics senza aspettare tutti i relatori — il vincolo «niente nomi non confermati» la rende sicura.

## R2 · Annuncio apertura iscrizioni

**Template**
```
Annuncio apertura iscrizioni di [EVENTO]. Pubblico: [PUBBLICO]. Leva: [LEVA: es. quota early prime 100]. Includi date, sede, link [LINK]. [N] versioni per [CANALI: LinkedIn / newsletter / sito]. Tono professionale; massimo 3 hashtag dove pertinenti.
```

## R3 · Newsletter post-evento

**Template**
```
Newsletter di chiusura per [EVENTO]: numeri dell’edizione ([PARTECIPANTI, SESSIONI, WORKSHOP: da verificare]), 3 momenti salienti ([QUALI]), ringraziamenti (faculty, sponsor senza gerarchie imbarazzanti, team), aggancio alla prossima edizione [SE ANNUNCIABILE]. Max 250 parole + oggetto in 3 varianti.
```

## R4 · Annuncio relatore confermato

**Template**
```
Post per annunciare la partecipazione di [RELATORE: nome, affiliazione, tema] a [EVENTO]. 2 righe sul perché il tema è rilevante per [PUBBLICO]. Verifica con me prima di pubblicare: nome e affiliazione esatti, conferma ricevuta. Versioni: LinkedIn + storia breve.
```
**Attenzioni:** l’annuncio di un relatore NON confermato è l’incidente di comunicazione più frequente del settore: il campo «conferma ricevuta» è un checkpoint, non burocrazia.
